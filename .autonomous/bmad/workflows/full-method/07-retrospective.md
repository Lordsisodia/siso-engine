# Workflow: 07 - Retrospective

**Phase:** Wrap-up
**Purpose:** Learn and improve
**Trigger:** ER (Epic Retrospective), SR (Sprint Retrospective)
**Agent:** Bob (SM) or any team member
**Complexity:** Low
**Time:** 30-60 minutes

## Overview

The Retrospective phase captures learnings:
- What went well
- What didn't go well
- Action items for improvement

## Input

- Completed work
- Metrics and data
- Team feedback

## Output

- `retro-{epic/sprint}-.md` - Retrospective document
- Action items

## Procedure

### Step 1: Gather Data

**Objective:** Collect facts

**Actions:**
1. Review completed work
2. Gather metrics
3. Note blockers encountered
4. Collect feedback

### Step 2: What Went Well

**Objective:** Celebrate successes

**Actions:**
1. List achievements
2. Note effective practices
3. Recognize contributions
4. Document wins

### Step 3: What Didn't Go Well

**Objective:** Identify problems

**Actions:**
1. List challenges
2. Note blockers
3. Identify process issues
4. Be specific

### Step 4: Root Cause Analysis

**Objective:** Understand why

**Actions:**
1. Pick top issues
2. Ask "why" 5 times
3. Identify patterns
4. Find systemic causes

### Step 5: Action Items

**Objective:** Commit to improvement

**Actions:**
1. Define specific actions
2. Assign owners
3. Set deadlines
4. Track in system

## Verification

- [ ] Data gathered objectively
- [ ] All voices heard
- [ ] Root causes identified
- [ ] Action items assigned
- [ ] Document shared

## Integration

Retrospectives feed into:
- Process improvements
- Next sprint planning
- Team practices
- BMAD workflow refinements
