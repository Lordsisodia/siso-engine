# Quick Flow: Phase 1 - SPEC

## Purpose

Rapid technical specification for small tasks. 5-10 minutes of planning before implementation.

## When to Use

- Bug fixes
- Small features
- Clear scope
- Single component
- Estimated < 2 hours

## Input

Task file from `/Users/shaansisodia/DEV/SISO-ECOSYSTEM/SISO-INTERNAL/blackbox5/5-project-memory/ralf-core/.autonomous/tasks/active/`

## Output

Quick Spec document in run folder

## Steps

### 1. Restate Goal

In one sentence, what are we solving?

```markdown
**Goal:** [Clear, concise statement]
```

### 2. Identify Scope

What's in scope? What's out?

```markdown
**In Scope:**
- [Item 1]
- [Item 2]

**Out of Scope:**
- [Item 1]
- [Item 2]
```

### 3. List Files to Modify

```markdown
**Files:**
- `/full/path/to/file1` - [what to change]
- `/full/path/to/file2` - [what to change]
```

### 4. Identify Tests

```markdown
**Tests Needed:**
- [ ] Existing test: [test name]
- [ ] New test: [what to test]
```

### 5. Assess Risk

```markdown
**Risk Level:** [Low | Medium | High]
**Rationale:** [Why this risk level]
```

## Quick Spec Template

```markdown
# Quick Spec: [TASK-ID]

**Goal:**

**In Scope:**

**Out of Scope:**

**Files to Modify:**

**Tests:**

**Risk:** [Level] - [Rationale]

**Estimated Time:** [X] minutes
```

## Success Criteria

- [ ] Goal clearly stated
- [ ] Scope defined (in/out)
- [ ] Files identified
- [ ] Tests noted
- [ ] Risk assessed
